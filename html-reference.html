A Valid HTML Page

All HTML pages must have a couple of things. The first is the <!DOCTYPE> tag, which lets the computer know that the file will hold HTML code. The <html> tag actually holds the code inside of it (see HTML Tag section). Inside the HTML tag are the <head> and <body> tags.

HTML Tag

Description

The HTML tag is used to communicate to the computer that the HTML code of the website will be inside of it.

Syntax

      <html>
        
        ...HTML code here...
        
      </html>

Example

The HTML tag must ALWAYS be used in order to program in HTML. It is used in the standard setup of the website, like so: 

      <!DOCTYPE>
      <html>
      <head<
      
      </head>
      <body<
      
      </body>
      <html>

Rendering

The html tag, of course, does not show up on the screen. It has no visual output; it is only meant to hold the code.

W3Schools link

Head Tag

Description

The <head> tag has many uses, all of which are the ability to use another tag within itself to give important information about the website, like a <title> or a <style> for the website.

Syntax

      <head>
      
        ...code...
        
      <head>

Example

The title of a website may be created inside of the head tag (see Title Tag section for more): 

      <head>
      <title> ... </title>
      </head>

Rendering

Most of the code put into the head tag will not show up on the finished website. However, there are a few exceptions, one of which, for example, is the <style> tag, which can change the overall look of the website.

**No W3Schools link

Title Tag

Description

The title tag adds a title to the website, which shows up not on the actual page, but instead on the tab at the top of the browser that the website is being viewed on.

Syntax

      <head>
      <title> ... </title>
      </head>

Example

To name a website "My Website," do the following inside of the head tag: 

      <head>
      <title> My Website </title>
      </head>

Rendering

Rather than appearing on the website, the title will be shown on the tab at the top of a browser.

**No W3Schools link

Body Tag

Description

The body tag holds most of the content, including the paragraphs, images, and most visible parts of the website.

Syntax

      <body>
      ...content...
      </body>

Example

One thing included in the body tag is all of the headings and paragraphs on your website: 

      <body>
        <h2>Heading</h2>
        <p>This is a paragraph.</p>
      </body>

Rendering

Everything created inside of the body tag will show up on the website, including things like headings, paragraphs, images, etc.

Paragraphs

Description

A paragraph can display any amount of any text; whether it is a sentence, or, of course, a paragraph.

Syntax

      <p>
        ...words...
      </p>

Example

A simple line of text is displayed from this code: 

      <p>
        This is a paragraph.
      </p>

Rendering

A paragraph will show up on the screen as whatever text has been entered into the tag.

Breaks

Description

A break is shown as an empty line, usually in between paragraphs.

Syntax

      <br>

Example

To add a break in between two paragraphs: 

      <p>
        ...paragraph 1...
      </p>
      <br>
      <p>
        ...paragraph 2...
      </p>

Rendering

A break shows up as an empty space in between two aspects of a website.

W3Schools link

Lines

Description

A line can be create in HTML. These may be used for clarifying the break between two sections of a website.

Syntax

      <hr>

Example

Divide two sections of a website: 

      <h2>Section 1</h2>
      <p>
        ...words...
      </p>
      <hr>
      <h2>Section 2</h2>
      <p>
      ...words...
      </p>

Rendering

It will show up as a horizontal line across the entire width of the screen.

Comments

Description

A comment, unlike anything that will show up on a website, was specifically created NOT to show up on a website. A comment is used by a website developer to help organize his/her code (mostly by labelling it).

Syntax

      <!-- (The comment goes here) -->

Example

Add a comment to organize your code, such as labelling paragraphs: 

      <!-- Paragraph 1 -->
      <p>
        ...paragraph...
      </p>

Rendering

A comment does not show up on a website.

Links

Description

A link can be used to redirect a website user to another website when they click on a certain couple of words or piece of text.

Syntax

      <a href="(website URL here)"> (Text that will show up) </a>

Example

Direct someone to Google: 

      <a href="https://www.google.com/">Google</a>

Rendering

The only thing that will show up on the screen is the text that you decide to display. In the example, that text was "Google".

W3Schools link

Images

Description

In HTML, an image may be added to a website. The width and height of the image may be controlled. An alternate string of text may be added in case the image is not able to be loaded.

Syntax

      <img src="(Image URL or file name)" width="(#)px" height="(#)px" alt="(alternate text)">

Example

Add an image of a cat: 

      <img src="cat.png" width="100px" height="200px" alt="Cat Picture">

Rendering

An image will show up on the website according to the height and width given. If none are given, it will default to the original size of the image. If something goes wrong that causes the image to not be loaded, the alternate text will show up where the image was supposed to be.

Lists

Description

Lists may be displayed on a website. There are two kinds: ordered and unordered. There are many ways of displaying ordered lists, as well, such as by using numbers, letters, or roman numerals.

Syntax

      Unordered list: <ul> ...list items... </ul>
      Ordered list: <ol> ...list items... </ol>
      List item: <li> ...list item text... </li>

Example

A grocery list: 

      <ul>
      <li> Milk </li>
      <li> Eggs </li>
      <li> Flour </li>
      <li> Sugar </li>
      </ul>

Rendering

The way that a list appears on a website depends on how it was programmed, including whether it was ordered or unordered, used numbers, letters, or roman numerals, etc. However, each list item is always on the next line.

W3Schools link

Tables

Description

A table is a way to organize information and can be displayed on a website with HTML.

Syntax

      Table: <table>...all table elements...</table>
      Table Row: <tr>...table data, headers, etc...</tr>
      Table Header: <th>...header name...</th>
      Table Data: <td>...text...</td>

Example

The code for a table of dates: 

      <table>
  <tr>
    <th>Day</th>
    <th>Month</th>
    <th>Year</th>
  </tr>
  <tr>
    <td>26</td>
    <td>May</td> 
    <td>2014</td>
  </tr>
  <tr>
    <td>29</td>
    <td>May</td>
    <td>2012</td>
  </tr>
</table>

Will display the following table:

Day Month Year
26 May 2014
29 May 2012

Rendering

The way a table looks is completely dependent on the code, plus styles may be added, so a table could look a large variety of different ways. However, the following applies to all tables:

  • All table rows display the text written inside the tags, from right to left, perfectly lined up underneath each header
  • All table headers, by default, are bold

Blocks

Discription

Tags such as <div> and <span> can be used to divide up portions of an element such as a pararaph, for example, in order to apply something (like a style) to only a part of it. <div> tags are used for blocks of code, and <span> tags are used within one line.

Syntax

  Div:  <div>
        ...block of code...
        </div>
        
  Span: <span>...a few words...</span>

Example

Applying style to a section of code: 

  <div style="color:blue">
    <p>
      This is paragraph 1.
    </p>
    <p>
      This is paragraph 2.
    </p>
  </div>

Applying to one word in a paragraph: 

  <p>
    This is a <span style="color:red">paragraph.</span>
  </p>

Rendering

By using <div> and <span> tags, you can change the way some text appears. The way it ends up looking depends on the code and what is changed about those specified sections of code/text.
